Reverse Engineering - ~50-100

Never opened a debugger? These are for you! Use IDA pro to analyze them(it's in the tool file!). You'll want to execute one of these on a linux box, at some point.

  • re50
  • re100

Hint for re50

Try looking at the strings menu in IDA to see interesting text in the file, you can double click them and check cross-references to see where they are used.

Hint for re100

Do you see the program flow branch off in IDA above the interesting strings? You can use gdb on linux(useful commands: "set disassembly-flavor intel", b *0xaddress, si, x/i $eip, set $eip=0xaddress) to control execution, or you can modify the assembly using a hex editor(a little harder! you've got to figure out the file offset of the instruction you want to change, and what the number is for the instruction you want to change it to using something like www.onlinedisassembler.com)

Exploitation - ~50

For the below challenge, you'll probably want a tool called "netcat." it's shortened to "ncat" or "nc" on certain linux distributions. apt-get install nmap if not. An example usage would be "ncat address port" to start a connection, you can type text in to send it off!

I've never done any security stuff before...

That's fine too! Register for picoctf here(don't need to give a valid email or anything): https://picoctf.com/registration

When you log in, you probably want to click "Switch to basic problem viewer" under their mini-game to see just the problems.

The key to solving them is thinking like a CTF player:

  • Do you see any phrases/words/numbers you might be able to google to try to get the answer? (try not to click on links that are writeups for the challenge! those don't exist for live CTFs)
  • Is the wording/phrasing/story of the challenge hinting at something?
  • Is there a tool I can find to do what I think I need to do? Google might know! (so will any of our helpers)
  • Go over what you know about the problem in your head, you might come up with an answer to one of the above.
